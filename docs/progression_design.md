# Progression Design

本文档记录 `Pantheon Age` 的成长系统设计。

目标不是做普通 RPG 的线性升级，而是做一个适合维多利亚神秘学、固定八神、职业组合、仪式晋升和道具代价的成长体系。

核心结构：

```text
职业等级：技能成长。
信仰等级：神恩成长。
仪式晋升：突破神秘阶位。
道具系统：外部力量和风险。
属性系统：统一检定标准。
代价系统：限制力量滥用。
```

一句话：

```text
职业决定你的手段。
信仰决定你的力量来源。
仪式决定你能走多深。
道具决定你能临时打破什么规则。
代价决定你还能不能安全回头。
```

## 设计原则

成长系统必须服务于项目核心方向：

```text
LLM creates possibilities.
Rules constrain authority, not imagination.
The rule system confirms reality.
Only validated structured state becomes game truth.
```

因此：

- LLM 可以提出技能表现、仪式画面、道具异象和晋升试炼；
- 规则系统决定是否升级、是否解锁能力、是否支付代价；
- 玩家不能因为一句自然语言输入就直接获得等级、天赋、祷告、圣物或阶位；
- 所有成长都必须以结构化状态保存。

## 总体成长结构

角色最终应包含这些成长字段：

```text
Character
  class_id
  class_level

  god
  faith_level
  devotion
  favor

  ascension_rank
  revelation

  attributes
  skills
  talents
  prayers
  items
  burdens
```

说明：

- `class_level`：职业等级，代表普通技能、训练和经验。
- `faith_level`：信仰等级，代表与某位神明秩序的连接深度。
- `devotion`：虔诚值，长期信仰倾向。
- `favor`：神眷值，可消耗资源，用于祷告或神恩。
- `ascension_rank`：神秘阶位，通过仪式突破。
- `revelation`：见证值，代表接触异常、完成事件和理解真相后的成长。
- `skills`：职业技能。
- `talents`：信仰天赋或组合天赋。
- `prayers`：可主动发动的祷告。
- `items`：普通道具、圣物、炼金物和诅咒物。
- `burdens`：代价、誓约、污染、债务、追踪标记或副作用。

## 职业等级

职业等级代表角色在“人类手段”上的成长。

职业等级主要提供：

- 主动技能；
- 被动技能；
- 检定加成；
- 特殊行动方式；
- 职业资源效率；
- 对特定场景的处理优势。

推荐初期使用 5 级：

```text
Lv1：职业基础能力。
Lv2：解锁第一个主动技能。
Lv3：获得职业被动或稳定加成。
Lv4：强化核心技能，降低失败代价。
Lv5：解锁职业标志能力。
```

职业等级不应该直接带来大幅全属性增长。

职业升级更应该体现：

```text
你多了一种解决问题的方式。
```

而不是：

```text
你的所有数值变大了。
```

### 职业技能例子

当前 Phase 8.2 已实现每个职业的 Lv1 签名技能基线。

这些技能的规则意义是：

- 技能会根据 Agent 裁定后的 `risk_type` 和 `check_stat` 提供小额加成；
- 技能来源会写进掷骰结果，例如 `技能：正面战斗基础 +2`；
- 技能只提高成功率，不会自动成功；
- 自然 1 仍然是严重失败，自然 20 仍然是完全成功；
- LLM 可以描写技能如何表现，但不能凭叙事直接授予成功、物品、线索、击杀或成长。

当前 Lv1 签名技能：

```text
骑士：正面战斗基础
法师：异常解析基础
密探：潜行开锁基础
游侠：追踪侦察基础
牧师：祷告仪式基础
炼金术士：药剂鉴定基础
```

骑士：

- Lv1：正面战斗检定获得基础加成。
- Lv2：守护，可以替目标承担一次伤害或风险。
- Lv3：威慑，面对普通士兵、混混或低阶怪物时获得交涉优势。
- Lv4：破阵，成功正面突破时降低额外风险。
- Lv5：誓约之盾，在重大危机中强行保护一个目标，但自己承担代价。

密探：

- Lv1：潜行、开锁或跟踪检定获得基础加成。
- Lv2：假身份，降低一次盘问或通行风险。
- Lv3：黑市人脉，更容易接触非法交易和情报源。
- Lv4：无声行动，潜入失败时可消耗资源避免立刻暴露。
- Lv5：无名者，短时间抹去自己的行动痕迹，但获得隐秘代价。

炼金术士：

- Lv1：药剂和材料鉴定获得基础加成。
- Lv2：快速调剂，可以在行动中使用或改造一次消耗品。
- Lv3：异常材料解析，调查毒素、圣物碎片和污染物时获得额外线索机会。
- Lv4：危险试剂，允许一次高风险高收益的炼金解决方案。
- Lv5：临场炼成，在危机中制造一次性特殊道具，但可能带来 HP、SAN 或污染代价。

## 信仰等级

信仰等级代表角色和某位神明秩序的连接深度。

信仰等级主要提供：

- 信仰天赋；
- 祷告；
- 神眷资源上限或恢复方式；
- 对应神明领域的感知能力；
- 禁忌、誓约和代价。

推荐初期使用 5 级：

```text
Lv1：信徒
Lv2：受礼者
Lv3：持誓者
Lv4：眷顾者
Lv5：代祷者
```

信仰等级不等于神秘阶位。

信仰等级回答：

```text
这位神明的秩序在你身上有多深？
```

神秘阶位回答：

```text
你整体上离超凡世界有多深？
```

### 信仰能力结构

每位神明提供两类能力：

```text
天赋：被动能力，长期生效。
祷告：主动能力，通常消耗 favor。
```

当前 Phase 8.3 已实现每位神明的 Lv1 天赋和 Lv1 祷告基线。

规则意义：

- 天赋是被动小加成，会根据 Agent 裁定后的 `risk_type` 和 `check_stat`
  匹配；
- 祷告是主动能力，必须在行动中被玩家表达出来，并消耗 `favor`；
- 祷告加成比天赋更明显，但神眷不足时不会生效；
- 在敌对或受限宗教环境中公开祷告，会增加怀疑风险；
- 天赋和祷告都不能直接授予线索、物品、旅行完成、击杀、阵营变化、
  结局或成长奖励。

当前 Lv1 信仰能力：

```text
海洋之神：潮汐感应 / 平潮祷告
真理之神：证词敏感 / 白塔明证
战争之神：军势直觉 / 铁血号令
审判之神：誓言感知 / 审判钟声
丰饶之神：生命嗅觉 / 蔷薇复苏
死亡之神：临终残响 / 安魂
隐秘之神：影中低语 / 无声祈祷
深渊之神：深渊回声 / 深渊低语
```

例子：

死亡之神：

- 天赋：临终残响。调查尸体、遗嘱、墓地、亡者痕迹时获得加成。
- 祷告：安魂。消耗神眷，尝试压制亡灵、降低恐慌或稳定 SAN。
- 禁忌：亵渎尸体、强行复活、阻止亡者安眠。

审判之神：

- 天赋：誓言感知。面对契约、证词和判决记录时，更容易发现违誓痕迹。
- 祷告：审判钟声。消耗神眷，短暂压制混乱异常或逼迫目标正面回应一个问题。
- 禁忌：伪誓、逃避合法审判、篡改判决记录。

隐秘之神：

- 天赋：影中低语。更容易识别暗号、假身份和被隐藏的通道。
- 祷告：无声祈祷。消耗神眷，短暂降低存在感或隐藏行动痕迹。
- 禁忌：公开同伴真实身份，把秘密卖给深渊势力。

## 仪式晋升体系

仪式晋升是系统中最重要的突破机制。

经验够了，不代表可以直接变强。

当前 Phase 8.5 已实现最小晋升切片：

- 职业等级：`class_level` 1 -> 2；
- 信仰等级：`faith_level` 1 -> 2；
- 神秘阶位：`ascension_rank` 0 -> 1。

当前规则：

- 职业晋升需要 `revelation >= 1`，消耗 1 点见证值，并给职业相关六属性 +1；
- 信仰晋升需要 `revelation >= 1` 和 `favor >= 1`，消耗 1 点见证值和 1 点神眷，并给 `devotion +1`；
- 第一次神秘晋升需要 `class_level >= 2`、`faith_level >= 2`、`revelation >= 3` 和 `favor >= 1`，成功后进入阶位 1 并获得一个 `burden`；
- 条件不足时，系统会返回结构化拒绝原因，例如 `revelation_not_enough`、`favor_not_enough`、`class_level_too_low`；
- LLM 可以描述训练、祷告和仪式画面，但不能直接提交等级、属性、资源或代价变化。

晋升必须满足：

```text
见证值 Revelation
职业等级要求
信仰等级要求
仪式媒介
特定事件
代价或誓约
```

推荐神秘阶位：

```text
阶位 0：凡俗者
阶位 1：见证者
阶位 2：受印者
阶位 3：执仪者
阶位 4：代行者
阶位 5：临座者
```

### 阶位说明

阶位 0：凡俗者

- 普通人。
- 可以拥有职业技能。
- 没有稳定神秘能力。

阶位 1：见证者

- 亲眼见过异常。
- 开始拥有轻微感知、抗性或异常经验。
- 仍然主要依赖职业手段。

阶位 2：受印者

- 被某位神明体系留下印记。
- 解锁第一个稳定信仰天赋。
- 容易被对应教会、秘社或异常注意到。

阶位 3：执仪者

- 可以主动举行低阶仪式或稳定祷告。
- 能把部分神明领域力量用于调查、净化、压制或探索。
- 代价开始明显。

阶位 4：代行者

- 可以短暂借用神明领域的力量改变局部现实。
- 会被教会、秘社、污染源或敌对神明体系重点关注。
- 强力能力必须伴随代价。

阶位 5：临座者

- 接近神明秩序边缘。
- 能影响小范围现实规则。
- 不能成神，也不能绕过世界核心规则。

### 晋升例子

从阶位 1 晋升阶位 2：

```text
要求：
- class_level >= 2
- faith_level >= 2
- revelation >= 100
- 完成一次与所信神明领域相关的重要事件
- 准备仪式媒介
- 接受一个誓约、副作用或长期关注
```

死亡之神晋升媒介：

- 无名墓土；
- 遗嘱残页；
- 黑烛；
- 丧钟碎片。

可能代价：

- 更容易听见亡者低语；
- 在墓地、停尸房和死亡现场触发额外 SAN 检定；
- 被安魂教团记录在案。

审判之神晋升媒介：

- 破损契约；
- 黑金墨水；
- 审判钟碎片；
- 违誓者的签名。

可能代价：

- 自己说出的誓言更难违背；
- 伪证、欺骗和逃避审判会带来更高代价；
- 被审判庭关注。

## 职业 + 信仰组合能力

`Pantheon Age` 的角色辨识度来自组合，而不是单一路线。

最终角色不只是：

```text
我是法师。
```

而应该是：

```text
我是 法师 + 真理之神 + 阶位 3 执仪者。
```

职业决定行动方式。

信仰决定力量气质。

阶位决定权限深度。

### 组合能力设计规则

组合能力应由三部分决定：

```text
职业动作类型 + 神明领域 + 当前阶位
```

例如：

- 骑士 + 审判之神：誓约、守护、决斗、压制混乱。
- 骑士 + 死亡之神：守墓、护送亡者、对抗亡灵、死亡决斗。
- 密探 + 隐秘之神：伪装、潜入、假身份、抹除痕迹。
- 密探 + 真理之神：审讯、证词分析、识破谎言、档案调查。
- 法师 + 真理之神：禁书解析、仪式推理、真相链追踪。
- 法师 + 深渊之神：禁忌知识、污染解析、高 SAN 风险收益。
- 牧师 + 丰饶之神：治疗、安抚、药草祝福、生命异变压制。
- 炼金术士 + 丰饶之神：药剂、肉体实验、医疗器械、瘟疫调查。
- 炼金术士 + 深渊之神：禁忌材料、污染实验、强效果高代价。
- 游侠 + 海洋之神：航线追踪、风暴预兆、殖民地远行。

### 组合能力限制

组合能力不能无限膨胀。

每个组合可以先只设计：

```text
1 个低阶天赋
1 个主动技能
1 个晋升后强化
1 个代价或禁忌
```

这样未来共有 `6 职业 * 8 信仰 = 48` 个组合，也不会一开始写爆。

## 道具系统

道具是外部力量。

它不应该只是：

```text
攻击 +5
```

而应该是：

```text
提供一次特殊规则权限，但伴随代价。
```

道具类型：

- 普通装备：剑、盾、火器、工具、衣物、证件。
- 消耗品：药剂、圣水、仪式粉末、镇静剂。
- 仪式媒介：墓土、契约碎片、海图、钟声残片。
- 圣物：教会认可的神明物品，有稳定规则和使用限制。
- 诅咒物：强效果、高代价、可能污染或吸引注意。
- 炼金造物：可制作、可改造、可失败。

当前 Phase 8.6 已实现最小道具切片：

- 背包仍以简单字符串列表保存，保证旧存档兼容；
- `phase1_cli/items.py` 为背包物品提供结构化 `category`、`effects` 和 `direct_use`；
- 玩家公开状态和 Agentic Runtime context pack 会暴露 `item_affordances`；
- 玩家明确调用已持有道具时，道具可以给 world-mode 检定提供 `item_bonuses`；
- 消耗品可以在检定或直接使用后从背包移除，并记录 `item_consumed`；
- 直接使用 `止血药剂`、`镇静药剂`、`小瓶圣水` 可以通过提交层改变 HP、SAN 或 Corruption；
- LLM 生成的临时物件仍不能直接进入背包，不能直接授予线索或状态变化。

这意味着道具现在是“可提交的规则资源”，不是 narrator 随口描述的奖励。
未来遗物、诅咒物和仪式媒介会复用同一套结构，只是效果和代价更复杂。

### 道具设计模板

每个重要道具应记录：

```text
name
category
description
effect
cost
uses
authority_level
allowed_contexts
forbidden_contexts
```

例子：

黑金审判铃：

- 类型：圣物 / 审判道具。
- 效果：在审讯、审判或契约场景中，强迫目标正面回应一个问题。
- 代价：如果问题本身不合法，Suspicion +1。
- 风险：如果目标受更高阶保护，玩家 SAN -1 或触发审判反噬。
- 权限：不能直接证明真相，只能迫使目标回应。

无名墓土：

- 类型：仪式媒介 / 死亡道具。
- 效果：用于安魂、追踪亡者残响或晋升仪式。
- 代价：携带期间更容易触发亡者低语事件。
- 权限：不能复活死者，不能凭空生成亡者真相。

被涂黑的通行证：

- 类型：隐秘道具。
- 效果：在一次通行、盘问或潜入场景中制造可信假身份。
- 代价：使用后可能被夜幕修会记录。
- 权限：不能绕过所有封锁，只能降低一次风险。

## 属性系统规划

当前 Phase 1 只有四个基础属性：

```text
strength      力量
agility       敏捷
intelligence  智力
faith         信仰
```

这对 CLI MVP 足够，但长期项目需要更细一点的属性，尤其要区分：

- 发现线索；
- 理解知识；
- 抵抗恐惧；
- 使用神明力量。

推荐最终使用 6 个核心属性：

```text
Physique   体魄
Agility    灵巧
Insight    洞察
Knowledge  学识
Will       意志
Communion  共鸣
```

中文字段可以用于展示，代码字段建议使用英文。

### 数值尺度

长期属性建议参考 d20 / BG3 式的小数值结构，但不直接复刻其职业、法术和专长体系。

推荐属性使用 `8-20` 作为常规范围：

```text
8-9     明显短板
10-11   普通人水平
12-13   受过训练
14-15   职业优势
16-17   罕见强项
18-19   高阶强项
20      常规上限
```

属性不要直接整值加入检定，而是先换算为 `attribute_modifier`：

```text
attribute_modifier = floor((attribute_score - 10) / 2)
```

示例：

```text
8-9    -1
10-11   0
12-13  +1
14-15  +2
16-17  +3
18-19  +4
20     +5
```

这样可以避免数值膨胀。

例如 `体魄 16` 不是给检定 `+16`，而是给检定 `+3`。

### 初始属性建议

角色创建时，普通初始范围建议是：

```text
最低属性：8
普通属性：10
职业主属性：14 或 15
极限初始强项：16
```

不要让初始角色轻易拥有 18 或 20。

推荐初始角色大致长这样：

```text
骑士：
  Physique 15
  Agility 10
  Insight 11
  Knowledge 9
  Will 14
  Communion 10

法师：
  Physique 8
  Agility 10
  Insight 12
  Knowledge 15
  Will 11
  Communion 14

密探：
  Physique 10
  Agility 15
  Insight 14
  Knowledge 11
  Will 10
  Communion 9
```

这些数值只代表方向，具体实现时可以允许玩家微调。

### 升级加属性

升级可以加属性，但不能每级都大量增加。

推荐规则：

```text
职业等级 2：选择 +1 一个职业相关属性，或学习一个职业技能。
职业等级 4：选择 +1 一个职业相关属性，或强化一个职业技能。
职业等级 5：优先解锁标志技能，不默认加属性。

信仰等级 2：选择 +1 Communion / Will，或学习一个祷告。
信仰等级 4：选择 +1 Communion / Will，或强化一个信仰天赋。

仪式晋升：通常不直接加大量属性，而是提高上限、解锁权限或增加代价。
```

也就是说，升级有成长感，但玩家要在这些选项中取舍：

```text
我要更高属性？
还是要新技能？
还是要更强祷告？
```

### 属性上限

推荐上限：

```text
普通训练上限：18
完成仪式晋升后：20
特殊圣物 / 诅咒物 / 高阶仪式临时上限：22
```

属性超过 20 应非常罕见，并且必须有明显代价。

例如：

- 深渊诅咒物临时让 `Knowledge +2`，但 SAN 检定难度提高；
- 战争圣物临时让 `Physique +2`，但战斗结束后 HP 或 Burden 增加；
- 审判仪式临时让 `Will +2`，但之后违背誓言代价翻倍。

### 六属性解释

体魄 Physique：

- 肉体强度、耐力、承伤、近战、搬运、破门。
- 影响 HP、负伤抗性、近战伤害和体力行动。

灵巧 Agility：

- 速度、潜行、开锁、闪避、远程武器、手上动作。
- 影响潜入、陷阱、远程命中、精准伤害和追踪中的行动精度。

洞察 Insight：

- 观察、直觉、读人、发现矛盾、现场调查。
- 影响找线索、识破谎言、注意异常细节和发现敌人弱点。

学识 Knowledge：

- 书本知识、神秘学、历史、医学、工程、炼金理论。
- 影响解析仪式、阅读禁书、制作药剂、理解技术装置。

意志 Will：

- 抗压、恐惧抵抗、精神稳定、坚持誓言。
- 影响 SAN、污染抵抗、梦境对抗和精神攻击抗性。

共鸣 Communion：

- 与神明、仪式、圣物和异常秩序的连接能力。
- 影响祷告、祝福、神谕感应、仪式发动。

### 为什么不是继续用四属性

四属性的问题：

- `intelligence` 同时承担“观察线索”和“理解知识”，太宽；
- `faith` 同时承担“信仰虔诚”和“精神抗性”，太宽；
- 潜入、社交、仪式和调查很难细分；
- LLM 生成复杂行动后，规则引擎缺少统一裁定维度。

六属性的好处：

- 洞察负责“我看没看出来”；
- 学识负责“我懂不懂这是什么”；
- 意志负责“我扛不扛得住”；
- 共鸣负责“我能不能向神秘秩序借力”。

### 派生属性

推荐派生属性：

```text
HP
SAN
Suspicion
Corruption
Favor
Revelation
Burden
```

HP：

- 生命和肉体状态。
- 主要受体魄、职业和装备影响。

SAN：

- 精神稳定。
- 主要受意志、职业、信仰和污染状态影响。

Suspicion：

- 社会层面的风险。
- 代表警察、教会、军方、秘社或当地势力注意到玩家。

Corruption：

- 神秘污染。
- 代表深渊、禁忌仪式、失败祷告、诅咒物和异常接触留下的痕迹。

Favor：

- 可消耗神眷。
- 用于祷告、祝福、神明领域相关行动。

Revelation：

- 见证值。
- 用于仪式晋升，不等于普通经验。

Burden：

- 代价集合。
- 可以包括誓约、债务、诅咒、被组织关注、名字模糊、亡者低语等。

### 属性与职业关系

职业提供初始倾向，但不锁死角色。

推荐方向：

- 骑士：体魄、意志较高。
- 法师：学识、共鸣较高，体魄较低。
- 密探：灵巧、洞察较高。
- 游侠：灵巧、洞察、体魄均衡。
- 牧师：意志、共鸣较高。
- 炼金术士：学识、洞察较高。

### 属性与信仰关系

信仰不应简单加固定属性。

信仰更适合提供：

- 对领域行动的加成；
- 特定场景的 advantage；
- 神眷恢复方式；
- 禁忌和代价。

例如：

- 海洋之神：航行、风暴、潮汐、港口事件获得领域加成。
- 真理之神：证词、档案、推理和真相揭露获得领域加成。
- 战争之神：纪律、威慑、战斗和军队事件获得领域加成。
- 审判之神：契约、誓言、法庭和审判事件获得领域加成。
- 丰饶之神：治疗、药剂、生命和瘟疫事件获得领域加成。
- 死亡之神：墓地、亡者、遗嘱和安魂事件获得领域加成。
- 隐秘之神：潜入、伪装、暗号和隐藏道路获得领域加成。
- 深渊之神：禁忌知识、污染解析和现实裂缝获得高风险加成。

## 检定建议

长期检定可以使用：

```text
d20 + attribute_modifier + proficiency_bonus + skill_bonus + context_bonus >= difficulty
```

其中：

- `attribute_modifier` 来自六属性换算；
- `proficiency_bonus` 来自角色总体熟练度；
- `skill_bonus` 来自职业等级和技能；
- `context_bonus` 来自信仰领域、道具、环境、身份或 NPC 关系；
- `difficulty` 由规则引擎决定；
- LLM 只能建议检定类型，不能决定成功。

当前 Phase 8.4 已实现 world-mode 六属性检定迁移基线：

- 旧四属性 `strength / agility / intelligence / faith` 暂时保留为兼容桥；
- world-mode 会根据 `risk_type + check_stat` 选择一个主要六属性；
- 主要六属性通过 `(attribute - 10) // 2` 转成 `attribute_modifier`；
- `attribute_modifier` 会进入 `行动修正`；
- 掷骰结果会记录 `attribute_profile`，玩家可见文本会显示类似
  `属性：体魄 15 +2`；
- 职业技能、信仰天赋、祷告和六属性修正现在走同一条 world-mode
  modifier 路径。

这个阶段不是最终数值平衡，而是把六属性正式接入实际裁定。

熟练加值建议保持很小：

```text
角色等级 / 成长期 1-2：+2
角色等级 / 成长期 3-4：+3
角色等级 / 成长期 5 或高阶晋升：+4
```

不要让多个熟练来源无限叠加。

如果角色已经有某技能熟练，再获得同类熟练，应该改为：

- 获得 expertise；
- 解锁特殊用法；
- 降低失败代价；
- 或转化为其他奖励。

Expertise 可以理解为“专精”：

```text
普通熟练：+proficiency_bonus
专精：+2 * proficiency_bonus
```

专精应非常有限，主要给密探、法师、炼金术士这类依赖专业行动的角色。

难度示例：

```text
easy      10
normal    14
hard      18
danger    22
mythic    26
```

参考判断：

- `normal 14`：普通训练者有机会成功；
- `hard 18`：需要好属性、熟练或有利环境；
- `danger 22`：需要强属性、熟练、道具或信仰帮助；
- `mythic 26`：必须有高阶能力、仪式、圣物或重大代价。

### 伤害与战斗数值

伤害不要只由一个属性决定。

推荐结构：

```text
damage = base_damage + relevant_attribute_modifier + skill_or_item_bonus
```

其中：

- `base_damage` 来自武器、祷告、炼金道具或怪物能力；
- `relevant_attribute_modifier` 来自行动方式；
- `skill_or_item_bonus` 来自职业技能、圣物、诅咒物或一次性道具。

不同伤害方式对应不同属性：

```text
普通近战伤害：Physique
远程 / 精准伤害：Agility
弱点 / 背刺伤害：Insight
炼金 / 技术伤害：Knowledge
精神压制 / 抵抗伤害：Will
祷告 / 仪式伤害：Communion
```

推荐基础伤害区间：

```text
徒手 / 临时武器：1d4
轻武器 / 小型道具：1d6
常规武器 / 稳定祷告：1d8
重武器 / 危险炼金物：1d10
高风险仪式 / 圣物爆发：2d6 或更高，但必须有代价
```

不要让普通升级直接让伤害翻倍。

伤害提升优先来自：

- 更好的武器；
- 更合适的技能；
- 对弱点的洞察；
- 信仰领域匹配；
- 道具消耗；
- 仪式代价。

### 防御与抗性

防御不建议只做一个大数字。

可以拆成：

```text
Guard      肉体防护，受装备、体魄和骑士技能影响。
Evasion    闪避能力，受灵巧和密探/游侠技能影响。
Resolve    精神抗性，受意志、牧师技能和信仰影响。
Warding    神秘防护，受共鸣、圣物和仪式影响。
```

第一版不一定全部实现，但长期设计上要避免所有防御都挤到同一个属性。

失败不一定只是“没做到”。

失败可以带来：

- HP 损失；
- SAN 损失；
- Suspicion 上升；
- Corruption 上升；
- 道具消耗；
- 事件升级；
- NPC 态度下降；
- 获得不完整或误导性信息；
- 触发代价。

## 与 LLM Runtime 的关系

LLM 可以生成：

- 技能表现；
- 仪式氛围；
- 道具异象；
- 晋升试炼；
- 祷告文本；
- 职业和信仰组合能力的叙事外观。

LLM 不能直接生成：

- 等级提升；
- 阶位提升；
- 技能解锁；
- 天赋解锁；
- 祷告解锁；
- 道具获得；
- 属性增加；
- HP / SAN / Favor / Revelation 变化；
- Burden 消除。

这些必须由规则系统授权，并写入结构化状态。

## 最小实现建议

不要一次实现完整系统。

推荐第一版只做：

```text
class_level
faith_level
revelation
favor
ascension_rank
```

以及：

- 每个职业 2 个技能；
- 每个神明 1 个天赋 + 1 个祷告；
- 只实现阶位 0 到阶位 2；
- 只实现少量仪式媒介；
- 只实现 3 到 5 个特殊道具；
- 暂时保留当前四属性，等 API 和存档迁移准备好后再升级到六属性。

这样可以先验证玩法，不会一次把代码改爆。
